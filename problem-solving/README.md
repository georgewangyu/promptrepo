## Core Concepts

### First Principles Thinking
Break problems down to fundamental truths and build up from there.

### Inversion
Instead of "how do I succeed?" ask "how do I guarantee failure?" Then avoid that.

### The 5 Whys
Keep asking "why?" until you hit root cause, not symptoms.

### Systems Thinking
Map inputs, outputs, feedback loops — see the machine, not just the parts.

## When to Use These

- Debugging persistent issues
- Understanding why something keeps happening
- Designing systems that won't break
- Finding creative solutions to stuck problems

## Quick Prompt

**For any stuck problem:**

```
I keep experiencing [problem]. 

Help me apply systems thinking:
1. Map the inputs and outputs of this system
2. What feedback loops might be reinforcing this?
3. What are the second-order effects I'm not seeing?
4. If I wanted to guarantee this problem continues, what would I do?
```
